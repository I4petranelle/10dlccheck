Overview

What the 10DLC compliance checker does

The extension reviews SMS content for common A2P 10DLC risk patterns (opt-in clarity, sensitive categories, pressure tactics, and misleading claims) and returns a simple status with suggestions. It does not expose carrier filtering rules.

Three ways to run a check

Opt-in assumed · Salesforce

A small “Check 10DLC” action appears in Salesforce message views. In this trusted context, Stop/Help lines can be hidden.

  • Works in Lightning and Classic
  • Designed for fast review while working a record
Opt-in unknown · Right-click

Highlight any text on a webpage, right-click, and select “Check 10DLC Compliance”. Useful for reviewing copy outside Salesforce.

  • Best for ad-hoc checks when opt-in context is unclear
Universal · Extension panel

Open the extension panel from the Chrome toolbar and paste content for quick review.

  • Great for drafting and tightening messages before sending

What you’ll see

  • Status: Pass · Warning · Fail
  • Issues: what triggered the status, in plain language
  • Tips: edits that improve clarity and reduce risk

FAQ

Common questions

Does the extension store or transmit message content? +

No. Checks run locally in your browser. Message content is not stored, logged, or transmitted to external servers.

Can I use it outside Salesforce? +

Yes. Use the right-click menu on any webpage or open the extension panel from the toolbar.

What if the extension stops working? +

Refresh the page, then toggle the extension in chrome://extensions. Restart Chrome and check for updates if needed.
